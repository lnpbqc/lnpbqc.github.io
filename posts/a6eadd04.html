1. SpringAI 框架概述

SpringAI是Spring官方推出的AI应用开发框架，它将大模型能力封装成Spring Boot风格，让Java开发者能够像使用其他Spring组件一样轻松集成AI功能。SpringAI的核心价值在于提供统一API抽象层，支持多种主流AI模型和服务，同时保持Spring生态的简洁性和一致性。

核心定位与优势：

  • 标准化接入：提供类似JdbcTemplate的模板方法调用LLM（大语言模型），降低学习成本
  • 多模型支持：统一接口支持OpenAI、Azure OpenAI、Ollama（本地模型）、Hugging Face等20+种AI服务
  • 企业级特性：内置重试机制、限流策略、监控指标等生产就绪功能
  • Spring生态集成：完美继承Spring Boot的自动配置特性，配置驱动，开箱即用

与Python生态的LangChain相对应，SpringAI为Java开发者提供了同等级别的AI应用开发体验，支持从简单的文本生成到复杂的RAG（检索增强生成）应用场景。

5. 工具调用（Function Calling）

5.1 什么是工具调用

大语言模型本质上是”纯大脑”——它们拥有海量知识，但存在两个天然局限：

局限 说明 举例
无法获取实时信息 模型的训练数据有截止日期，不知道”现在几点”、”今天什么日期” 问”明天星期几？”模型只能猜测
无法与外部世界交互 模型不能操作文件系统、调用API、访问数据库、控制设备 问”帮我订个闹钟”，模型只能口头建议

工具调用（Function Calling）就是解决这两个问题的机制：你定义好可执行的Java方法作为”工具”，模型在需要时自动决定调用哪个工具，SpringAI代为执行并将结果返回给模型，最终模型结合工具结果生成准确回答。

执行流程：

关键点：调用哪个工具、传什么参数、何时调用，全部由模型自主决定——你只需要把工具注册进去。

5.2 @Tool 注解详解

SpringAI 通过 @Tool 注解将普通 Java 方法标记为可供模型调用的工具。注解的核心要素：

要素 作用 示例
description 最重要——模型根据描述决定是否调用该工具。描述越清晰准确，调用越可靠 "Get the current date and time in the user's timezone"
方法名 作为工具名称暴露给模型 getCurrentDateTime
参数类型与名称 模型自动推断需要传什么参数 String time → 模型会传入时间字符串
返回值 工具执行结果，会作为上下文返回给模型 String → 模型可理解的自然语言描述

编写技巧：description 用英文写效果通常更好（当前主流模型的工具选择对英文更敏感），返回值也建议返回模型易于理解的字符串而非原始对象。

5.4 @ToolParam：为工具参数添加描述

当工具方法的参数不止一个时，模型需要知道每个参数的含义才能正确传参。@ToolParam 注解就是用来描述方法参数的：

注解 作用位置 用途
@Tool 方法 标记方法为可调用工具，description 告诉模型这个工具能做什么
@ToolParam 方法参数 描述每个参数的含义，告诉模型这个参数该填什么

当方法只有一个参数且参数名足够自解释时，@ToolParam 可以省略；多参数场景强烈建议加上，否则模型可能把内容填到文件名参数里。

5.6 工具调用的最佳实践

实践 说明
description 是灵魂 模型靠它决定何时调用。写清楚工具做什么、何时用、参数格式
一个工具只做一件事 readFile + writeFile 分开，而不是一个巨大的 doFileOperation 方法
返回值要”模型友好” 返回可读的字符串而非 JSON 对象，模型理解得更好
工具类尽量无状态 每次调用应该是独立的，避免成员变量跨请求污染
注意安全边界 文件操作务必做路径校验（normalize + startsWith），不要盲目信任模型的输入
多参数必须用 @ToolParam 两个以上同类型参数时，@ToolParam(description) 是模型区分它们的唯一线索
异常即反馈 抛出的异常会被模型读到——用它向用户解释”为什么操作失败”
