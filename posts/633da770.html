什么是 OpenFeign？

OpenFeign 是 Spring Cloud 官方提供的 声明式 HTTP 客户端（Declarative HTTP Client）。

它最大的特点就是：

像调用本地方法一样调用远程服务。

现在：

开发者只需要定义接口，不需要关心：

  • URL拼接
  • HTTP请求发送
  • JSON转换
  • 服务发现
  • 负载均衡

全部由 OpenFeign 自动完成。

OpenFeign = HTTP客户端 + 服务发现 + Ribbon(旧版本)/Spring Cloud LoadBalancer + 接口代理。

OpenFeign 的优点

相比 RestTemplate：

✔ 代码更加简洁

✔ 面向接口编程

✔ 自动服务发现

✔ 自动负载均衡

✔ 支持请求拦截器

✔ 支持超时配置

✔ 支持日志

✔ 支持降级(Fallback)

✔ 与 SpringMVC 注解完全兼容

OpenFeign 最佳实践

  1. Feign 接口建议单独抽取到 api 模块，方便多个服务复用。
  2. @FeignClient(name = "...") 中的 name 必须与注册中心服务名一致。
  3. 接口路径、HTTP 方法、参数注解要与服务提供者完全一致。
  4. 优先使用 FallbackFactory，便于获取异常信息和记录日志。
  5. 根据业务场景合理设置 connectTimeout 与 readTimeout，避免请求长时间阻塞。
  6. 开发阶段可开启 FULL 日志，生产环境建议使用 BASIC 或关闭日志，减少性能开销。
  7. 不建议在 Feign 接口中编写业务逻辑，Feign 仅负责远程调用。
  8. 在 Spring Cloud 2020+ 版本中，OpenFeign 默认集成 Spring Cloud LoadBalancer 实现客户端负载均衡；旧版本则默认集成 Ribbon。
